• Start wpa_supplicant on most systems by doing this: 
    # NOTE: check to see what wifi interfaces you have first
ip link show

# Start wpa_supplicant and observe results
sudo wpa_supplicant -i wlp3s0 -c /etc/wpa_supplicant/wpa_supplicant.conf

# NOTE: If you want to fork to background automatically then do this:
sudo wpa_supplicant -i wlp3s0 -c /etc/wpa_supplicant/wpa_supplicant.conf -B
  • Running wpa_supplicant on boot (disabling NetworkManager):
  • NOTE: I use wpa_supplicant entirely to get around college WPA2-Enterprise network
    • I should probably resort to a more flexible solution in the future (IE: NM) 
      # See if NetworkManager is enabled
systemctl status NetworkManager
# If enabled then disable:
sudo systemctl stop NetworkManager
sudo systemctl disable NetworkManager
# There are a couple other NetworkManager services I can't think of right 
# now that need to be disabled as well

# See if wpa_supplicant is enabled
systemctl status wpa_supplicant
# If not then enable:
sudo systemctl start wpa_supplicant
sudo systemctl enable wpa_supplicant

# We need a DHCP client to assign us a IP-address
# Check if dhcpcd is enabled
systemctl status dhcpcd
# If not then enable:
sudo systemctl start dhcpcd
sudo systemctl enable dhcpcd
  • Run wpa_supplicant on boot (via systemd)
  • Here's my "/etc/systemd/system/wpa_supplicant_init.service" 
    [Unit]
Description=WPA Supplicant Initialization

[Service]
Type=forking
ExecStart=/etc/wpa_supplicant/wpa_supplicant.conf

[Install]
WantedBy=multi-user.target
  • Run via systemd: 
    sudo systemctl start wpa_supplicant_init.service
sudo systemctl enable wpa_supplicant_init.service
# Check to see if it errored:
systemctl status wpa_supplicant_init.service
# Fix errors accordingly
